# 设计原则

## UNIX/LINUX设计原则

### 小即是美

### 让每个程序只做好一件事

### 快速建立原型

### 舍弃高效率而取可移植性

### 采用纯文本来存储数据（人可读性）

### 充分利用软件的杠杆效应（软件复用）

### 使用shell脚本来提高杠杆效应和可移植性

### 避免强制性的用户界面（用户界面和系统分开）

### 每个程序都称为过滤器

##  SOLID五大设计原则

### S-单一职责原则

- 一个程序只做好一件事
- 如果功能过于复杂就拆分开，每个部分保持独立

### O-开放封闭原则

- 对扩展开放，对修改封闭

### L-李氏置换原则

- 子类能够覆盖父类
- 父类能够出现的地方子类就能出现

### I-接口独立原则

- 保持接口单一独立，避免“胖接口”

### D-依赖倒置原则

- 面向接口编程，依赖于抽象而不依赖于具体
- 适用方只关注接口而不关注具体类的实现
